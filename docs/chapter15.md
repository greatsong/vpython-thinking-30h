### 프로젝트 A: 태양계 시뮬레이션 ☀️

태양 주위를 행성들이 돌고 있습니다. 슬라이더로 시간 속도를 조절하면 행성이 빠르게 또는 느리게 공전합니다.

- **핵심 개념** — 삼각함수(cos/sin), for+리스트, 함수, 슬라이더
- **성격** — 우주·천문·수학

👉 [Ch.15-A 바로가기](chapter15-1.md)

### 프로젝트 B: 먹고 먹히는 구 게임 🟢

구를 키보드로 조종합니다. 먹이를 먹으면 커지고, 적과 만나면 — 큰 쪽이 작은 쪽을 먹습니다! 나보다 큰 적은 피하고, 작은 적은 잡아먹는 긴장감 넘치는 게임.

- **핵심 개념** — 키보드(keysdown), 충돌(mag), 조건문, 리스트, 랜덤
- **성격** — 게임·경쟁·전략

👉 [Ch.15-B 바로가기](chapter15-2.md)

### 프로젝트 C: 3D 수족관 🐠

물고기들이 3D 수조 안에서 자유롭게 헤엄칩니다. 버튼으로 먹이를 주면 물고기가 모여들고, 바닥에서 거품이 올라옵니다. 보고만 있어도 힐링!

- **핵심 개념** — 3D 이동(x,y,z), 함수, 버튼, 슬라이더, 랜덤
- **성격** — 자연·힐링·예술

## 📋 어떤 프로젝트든 통하는 설계법

코드를 쓰기 전에 **설계**부터 합시다. 어떤 프로젝트를 고르든, 이 세 가지 질문에 답하면 설계가 끝납니다.

### 질문 1: 무엇이 움직이나요?

예: 행성이 태양 주위를 돈다 / 구가 먹이를 쫓는다 / 물고기가 헤엄친다

### 질문 2: 사용자가 뭘 조절하나요?

예: 슬라이더로 속도 조절 / 키보드로 이동 / 버튼으로 먹이 주기

### 질문 3: 어떤 규칙이 적용되나요?

예: 원 궤도를 따른다 / 큰 구가 작은 구를 먹는다 / 벽에 닿으면 방향 전환

이 세 가지를 정리했으면 설계 끝! 이제 코드로 옮기면 됩니다.

### 어떤 도구를 쓸지 체크하기

내 프로젝트에 필요한 것들을 확인하세요.

- [ ] **물체 만들기**: sphere, box, cylinder, cone 중 어떤 것?
- [ ] **변수**: 속도, 각도, 점수, 상태 등
- [ ] **랜덤**: 위치, 색상, 크기에 무작위 요소가 필요한가?
- [ ] **애니메이션**: while True + rate()로 매 프레임 업데이트
- [ ] **조건문**: 특정 상황에서 동작을 바꾸는가?
- [ ] **키보드**: keysdown()으로 사용자가 직접 조종하는가?
- [ ] **충돌 감지**: mag()로 물체 사이 거리를 재는가?
- [ ] **for + 리스트**: 비슷한 물체를 여러 개 만드는가?
- [ ] **함수**: 반복되는 코드를 묶을 수 있는가?
- [ ] **위젯**: slider, button으로 사용자가 조절할 수 있는가?
