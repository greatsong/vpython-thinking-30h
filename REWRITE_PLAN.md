# VPython 사고력 교재 — 전면 재작성 계획 (확정)

> **이 파일이 재작성의 유일한 진실 공급원(Single Source of Truth)이다.**
> 새 세션에서는 이 파일만 읽으면 바로 이어서 작업할 수 있다.
> 방향성에 대한 질문은 하지 않는다 — 이미 사용자와 정렬 완료.

## 1. 핵심 방향성 (사용자 확인 완료)

### 비판과 대응

| 비판 | 원인 | 대응 |
|------|------|------|
| 사고력이 안 키워짐 | 🧠 챌린지가 챕터 끝 5%에만 | 사고력 활동 4곳 분산 (~34%) |
| 시각화 미강조 | MkDocs 기본 테마, 이미지 0개 | GlowScript iframe + Mermaid + CSS 블록 |
| 문법책 느낌 | 📚개념(50%)→🔨적용(30%) 구조 | 🎬결과→🔍예측→🛠️탐구 구조로 반전 |
| 너무 어려움 | 챕터당 5-6개념, 코드 30-40줄 | 최대 3개념, 코드 15줄 이하 |

### 확정된 원칙

1. **결과물 먼저**: 모든 챕터는 완성된 3D 장면을 먼저 보여주고 역추적
2. **문법 25-30%**: 나머지는 사고력 활동(34%) + 실습(~36%)
3. **코드 길이 짧게, 예제 수 풍부하게**: 개별 블록 15줄 이하, 실험/예제는 다양하게
4. **개념 최대 3개/챕터**: 과부하 방지
5. **점진적 확장**: 단순→복잡으로 단계별 문제 해결
6. **GlowScript 기반**: 모든 코드 첫 줄 `GlowScript 3.2 VPython`
7. **마크다운 테이블 금지**: Mermaid, 볼드+목록, admonition으로 대체

### 4가지 사고 훈련 유형 (사용자 "대체로 맞음" 확인)

| 아이콘 | 유형 | 설명 | 예시 |
|--------|------|------|------|
| 🔍 | 코드→결과 예측 | 코드를 읽고 3D 결과를 종이에 그림 | `vector(i, i, 0)` → 대각선 |
| 🔄 | 결과→코드 역추적 | 장면 설명을 보고 코드 작성 | "구 5개 일직선" → for문 |
| 🐛 | 버그 사냥 | 고장난 코드를 찾아 고침 | 공이 벽을 뚫는 코드 |
| 💡 | 상상→코드 창작 | 제약 조건 안에서 자유 설계 | "구 3개로 눈사람" |

## 2. 챕터 구조 (확정)

### 15챕터 × 2시간 = 30시간

| # | 파트 | 제목 | 핵심 개념 (최대 3개) | 사고력 포인트 |
|---|------|------|---------------------|-------------|
| 1 | Part 1: 3D 첫 걸음 | VPython의 세계로! | sphere, color | 한 줄 바꾸면 뭐가 달라지나 |
| 2 | | 좌표의 비밀 | vector, pos | 좌표→위치 공간추론 |
| 3 | | 다양한 물체, 다양한 속성 | box/cylinder/cone, radius/size | 물체마다 속성 패턴 발견 |
| 4 | Part 2: 변화와 움직임 | 변수와 계산 | 변수, 연산, .pos 접근 | 이름이 없으면 조종 불가 |
| 5 | | **랜덤의 마법** | random(), uniform() | 같은 코드, 매번 다른 결과 |
| 6 | | 움직여라! 애니메이션 | while True, rate(), 위치 업데이트 | 속도를 바꾸면 어떻게? |
| 7 | Part 3: 게임을 향해 | 벽에 부딪히면? if문 | if, 비교연산 | 언제 방향을 바꿔야 하나 |
| 8 | | 키보드로 조종하기 | keysdown() | 입력→동작 매핑 설계 |
| 9 | | 충돌 감지 | mag(), 거리 판단 | "만났다"를 수학으로 표현 |
| 10 | | **미니 프로젝트: 나만의 게임** | Ch.4-9 종합 | 설계→분해→구현 |
| 11 | Part 4: 패턴과 구조 | for 반복문: 패턴의 힘 | for, range() | 1D→2D→3D 패턴 확장 |
| 12 | | 리스트: 물체를 모아서 | list, append, for+list | 개별 vs 그룹 관리 |
| 13 | Part 5: 설계와 완성 | 함수: 나만의 명령어 | def, 매개변수 | 나무→숲→산 추상화 |
| 14 | | 위젯과 대시보드 | slider, button | 다양한 상황의 UI 설계 |
| 15 | | 최종 프로젝트 | 전체 종합 | 설계→분해→구현→테스트 |

### 주요 결정 사항

- **랜덤을 Ch.5로 배치** (사용자 요청: "4~10챕터 사이에"). 변수(Ch.4) 직후라 자연스럽고, 시각적으로 즉시 재미있음
- **while+물리 별도 챕터 삭제**: while은 Ch.6(애니메이션)에서 자연 도입, 물리(중력/바운스)는 Ch.7/Ch.10에서 활용
- **미니 프로젝트(Ch.10) 신설**: Part 3 끝에 절반 지점 성취감
- **변수+연산 통합(Ch.4)**: 각각 2시간은 얇아서 합침
- **챕터 수 15개 유지**: 사용자 "늘어도 줄어도 됨" → 현재 15개가 균형 적절

### 점진적 확장 패턴 (사용자 확인 완료)

| 챕터 | 단계1 (단순) | 단계2 (확장) | 단계3 (도전) |
|------|------------|------------|------------|

## 3. 챕터 섹션 템플릿 (확정)

```
1. 🎬 오늘의 장면        (~5%)   결과물을 먼저 보여줌 + GlowScript 데모 iframe
2. 🔍 코드 읽기 챌린지    (~8%)   [사고력] 코드→결과 예측
3. 🛠️ 직접 만들어보기    (~20%)  탐구형 실습 (실험 2-3개, 점진적 확장)
4. 🔄 역추적 챌린지       (~8%)   [사고력] 결과→코드 역추적
5. 📖 알고 넘어가기       (~12%)  문법 정리 (1-2개 admonition 박스)
6. 🐛 버그 사냥           (~8%)   [사고력] 고장난 코드 고치기
7. 💡 상상 챌린지         (~10%)  [사고력] 자유 창작 (제약 조건 제시)
8. 📝 오늘의 완성 코드    (~10%)  전체 코드 + GlowScript 데모 iframe
9. ✅ 3줄 정리            (~4%)   핵심 3문장 요약
10. 🚀 더 탐험하기        (~15%)  선택형 확장 과제
```

사고력 활동: 섹션 2, 4, 6, 7 = 4곳 (~34%)
문법: 섹션 5 = 1곳 (~12%, admonition 합쳐 25-30%)

## 4. 시각화 전략 (확정)

### 3단계 시각화

**1단계: GlowScript 라이브 데모** (필수)
- 챕터의 🎬 오늘의 장면 + 📝 완성 코드에 iframe 임베드
- `docs/demos/ch01_scene.html`, `docs/demos/ch01_final.html` 형태
- GlowScript Export로 standalone HTML 생성
- 챕터당 2개 × 15챕터 = 약 30개

**2단계: Mermaid 다이어그램** (선택적)
- 프로그램 흐름도 (Ch.6 애니메이션 루프, Ch.7 if 분기, Ch.11 for 흐름)
- 적합한 챕터에만 사용

**3단계: HTML/CSS 블록** (보조)
- 코드 결과 미리보기 (어두운 배경 + 컬러 원/사각형)
- 좌표계 다이어그램
- Material admonition: tip(실험), bug(버그), question(힌트), success(정답)

### 접기/펼치기 규칙

```markdown
??? question "질문 텍스트"
    정답 (접혀 있음, 클릭하면 펼침)

!!! tip "바꿔 보기"
    실험 안내 (기본 펼쳐져 있음)

!!! bug "버그를 찾으세요!"
    고장난 코드

??? success "정답 예시"
    코드 + 설명
```
