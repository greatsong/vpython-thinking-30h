## 프로젝트 개요

### 핵심 철학
**"생각을 코드로, 코드를 3차원으로"**

- **컴퓨팅 사고력(CT)**이 궁극적 목표. 코딩은 도구
- **GlowScript** (glowscript.org) 기반 수업
- 매 챕터 4가지 사고 훈련: 예측(🔍), 역추적(🔄), 디버깅(🐛), 창작(💡)
- **결과물을 먼저 보여주고 역추적**하는 구조

### 대상
프로그래밍 경험 없는 고1, 30시간(15회 × 2시간)

---

## 챕터 구조 (10개 섹션)

```
1. 🎬 오늘의 장면        (~5%)   결과물 먼저 + GlowScript 데모
2. 🔍 코드 읽기 챌린지    (~8%)   [사고력] 코드→결과 예측
3. 🛠️ 직접 만들어보기    (~20%)  탐구형 실습 (점진적 확장)
4. 🔄 역추적 챌린지       (~8%)   [사고력] 결과→코드 역추적
5. 📖 알고 넘어가기       (~12%)  문법 정리 (admonition 1-2개)
6. 🐛 버그 사냥           (~8%)   [사고력] 고장난 코드 고치기
7. 💡 상상 챌린지         (~10%)  [사고력] 자유 창작
8. 📝 오늘의 완성 코드    (~10%)  전체 코드 + GlowScript 데모
9. ✅ 3줄 정리            (~4%)   핵심 3문장 요약
10. 🚀 더 탐험하기        (~15%)  선택형 확장 과제
```

## 코드 규칙

- 모든 코드 첫 줄: `GlowScript 3.2 VPython`
- glowscript.org에서 바로 붙여넣기 가능
- 코드 블록당 최대 15줄, 예제 수는 풍부하게
- 챕터당 새 개념 최대 3개
- WHAT/WHY 주석은 📝 완성 코드(섹션8)에만
- **마크다운 테이블 금지** — admonition, 볼드+목록으로 대체

---

## 시각화 규칙

1. **GlowScript iframe**: 🎬 오늘의 장면 + 📝 완성 코드에 3D 데모 임베드
2. **Mermaid**: 프로그램 흐름도에 선택적 사용
3. **HTML/CSS 블록**: 코드 결과 미리보기, 좌표 다이어그램
4. **Material admonition**: tip(실험), bug(버그), question(힌트), success(정답)

### 접기/펼치기

```markdown
??? question "질문"
    정답 (접혀 있음)

!!! tip "바꿔 보기"
    실험 안내 (펼쳐져 있음)

!!! bug "버그를 찾으세요!"
    고장난 코드

??? success "정답 예시"
    코드 + 설명
```

## 점진적 확장 원칙

모든 챕터에서 "단순 → 확장 → 도전" 3단계로 진행.
한 번에 완성품을 주지 않고, 단계마다 하나씩 문제를 추가한다.
